# Skill · Repurpose — 1 pieza ancla → N derivados

> **Agente:** PRISMA · **Input:** una pieza ancla terminada (video, episodio, artículo) · **Output:** set de derivados especificados por canal, en `brand/content/`.

## Pasos

1. **Verifica la etapa** (`state.md`): los canales de destino deben estar activos en la etapa del usuario. En E0-E1, frena el multicanal y dilo.
2. **Descompone el ancla en átomos:** secciones, takeaways, datos, frases citables, momentos punchy (el mejor minuto, la mejor historia).
3. **Mapea átomo → formato → canal** con `core/playbooks/platforms.md`. Referencia: 5-10 verticales + 1-2 carruseles + 2-4 posts + 1 newsletter + quotes. **Si el ancla da para menos, di la verdad:** 3 derivados buenos > 10 mediocres.
4. **Especifica cada derivado:** corte exacto (min X:XX–Y:YY o párrafo) · hook propio · texto en pantalla · CTA · ajuste de canal (9:16 y subtítulos en verticales; IG sin marca de agua; LinkedIn texto/carrusel con link al 1er comentario, nunca el video). Marca **[directo]** o **[requiere copy nuevo]**.
5. **Entrega el set** con calendario sugerido → TEMPO lo distribuye. Guarda en `brand/content/`.

## Formato de salida

```
ANCLA: [título · tipo]
1. VERTICAL · "hook" · corte X:XX–Y:YY · texto pantalla · CTA · → YT/IG/TikTok · [directo]
2. CARRUSEL · tema · N slides · → LinkedIn doc + IG · [requiere copy nuevo]
3. POST · idea · hook línea 1 · link→1er comentario · [requiere copy nuevo]
4. NEWSLETTER · ángulo de profundización
5. QUOTE · "…"
CALENDARIO SUGERIDO → TEMPO
```

## Anti-patrones
Mismo archivo a todos lados · derivado que no se sostiene solo · forzar volumen desde un ancla floja · derivar hacia canales que la etapa no tiene activos.
